# Product Service

Manages product catalog with PostgreSQL storage and Elasticsearch for vector-based semantic search.

## Overview

| Property | Value |
|----------|-------|
| Port | 9002 |
| Database | PostgreSQL (`products` table) |
| Search | Elasticsearch (vector embeddings) |

## Responsibilities

1. **Product CRUD**: Create, read, update products
2. **Product Listing**: List all products with pagination
3. **Vector Search**: Semantic product search using embeddings
4. **Elasticsearch Sync**: Sync products to Elasticsearch for vector search

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/products` | List all products |
| GET | `/api/products/{id}` | Get product by ID |
| POST | `/api/products/batch` | Get multiple products by IDs |
| POST | `/api/products/vector-search` | Semantic search using vector embeddings |

## Data Model

### Products Table

```sql
CREATE TABLE products (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    price NUMERIC(10, 2) NOT NULL,
    stock_quantity INT NOT NULL DEFAULT 0,
    metadata JSONB,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_products_metadata ON products USING GIN (metadata);
```

### Product Metadata Schema (JSONB)

```json
{
  "category": "shirt",
```

## Elasticsearch Integration

- **Index**: Products indexed with text fields + embedding vector
- **Sync**: `ElasticsearchSyncService` syncs products on create/update
- **Search**: Uses vector similarity for semantic matching

### ProductDocument (Elasticsearch)

```java
@Document(indexName = "products")
public class ProductDocument {
    @Id
    private Long id;
    @Field(type = Text)
    private String name;
    @Field(type = Text)
    private String description;
    @Field(type = Keyword)
    private String category;
    @Field(type = Keyword)
    private String brand;
    @Field(type = Keyword)
    private List<String> colors;
    @Field(type = Keyword)
    private List<String> sizes;
    @Field(type = Text)
    private String searchText; // Combined text for search
}
```

## Development Notes

- Products stored in PostgreSQL for persistence
- Elasticsearch used for semantic vector search only
- `metadata` JSONB column allows flexible product attributes
- Embeddings generated by AI layer, stored in metadata
